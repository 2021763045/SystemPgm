## 네트워크(Network)

### 1. 개요

네트워크(Network)는 두 대 이상의 컴퓨터나 장치가 서로 연결되어 데이터와 자원을 공유하는 시스템이다. 네트워크를 통해 파일 공유, 인터넷 사용, 메시지 전송 등이 가능하다.

---

### 2. 네트워크의 구성 요소

| 구성 요소         | 설명                     |
| ------------- | ---------------------- |
| 호스트(Host)     | 네트워크에 연결된 컴퓨터 또는 장치    |
| 서버(Server)    | 서비스를 제공하는 컴퓨터          |
| 클라이언트(Client) | 서비스를 요청하는 컴퓨터          |
| 라우터(Router)   | 서로 다른 네트워크를 연결하는 장치    |
| 스위치(Switch)   | 같은 네트워크 내 장치들을 연결하는 장치 |

---

### 3. IP 주소

IP 주소는 네트워크에서 각 장치를 구분하기 위한 고유한 주소이다.

예시

```text
192.168.0.10
```

특징

* 인터넷 통신 시 필수
* IPv4와 IPv6 사용
* 장치 식별 역할 수행

---

### 4. 포트(Port)

포트는 하나의 컴퓨터에서 여러 네트워크 서비스를 구분하기 위한 번호이다.

주요 포트 번호

| 포트     | 서비스   |
| ------ | ----- |
| 20, 21 | FTP   |
| 22     | SSH   |
| 25     | SMTP  |
| 53     | DNS   |
| 80     | HTTP  |
| 443    | HTTPS |

---

### 5. TCP와 UDP

#### TCP (Transmission Control Protocol)

* 연결 지향 방식
* 데이터 신뢰성 보장
* 순서 보장
* 속도는 상대적으로 느림

활용 예시

* 웹 서비스
* 이메일
* 파일 전송

#### UDP (User Datagram Protocol)

* 비연결 방식
* 빠른 전송 속도
* 신뢰성 보장 없음
* 데이터 손실 가능

활용 예시

* 실시간 스트리밍
* 온라인 게임
* 화상 회의

---

### 6. OSI 7계층

네트워크 통신 과정을 7단계로 구분한 모델이다.

| 계층                    | 역할          |
| --------------------- | ----------- |
| 7계층 응용(Application)   | 사용자 서비스 제공  |
| 6계층 표현(Presentation)  | 데이터 형식 변환   |
| 5계층 세션(Session)       | 연결 관리       |
| 4계층 전송(Transport)     | TCP, UDP 통신 |
| 3계층 네트워크(Network)     | IP 주소 및 라우팅 |
| 2계층 데이터 링크(Data Link) | MAC 주소 사용   |
| 1계층 물리(Physical)      | 실제 데이터 전송   |

---

### 7. 주요 네트워크 명령어

| 명령어          | 설명                 |
| ------------ | ------------------ |
| `ping`       | 네트워크 연결 확인         |
| `ifconfig`   | 네트워크 정보 확인 (Linux) |
| `ip addr`    | IP 주소 확인           |
| `netstat`    | 네트워크 상태 확인         |
| `traceroute` | 패킷 이동 경로 확인        |

예시

```bash
ping google.com
```

→ 네트워크 연결 상태 확인

---

### 정리

| 항목      | 설명         |
| ------- | ---------- |
