## Linux 파일 시스템 정리

### 1. 개요

Linux 파일 시스템은 파일과 디렉터리를 계층적 구조(Tree Structure)로 관리하는 시스템이다. 모든 파일과 디렉터리는 최상위 디렉터리인 루트(`/`) 아래에 위치한다.

---

### 2. 파일 시스템 구조

Linux는 트리 형태의 디렉터리 구조를 사용한다.

```text
/
├── bin
├── boot
├── dev
├── etc
├── home
├── tmp
├── usr
└── var
```

특징

* 루트 디렉터리(`/`)가 시작점
* 모든 파일과 디렉터리가 루트 아래에 존재
* 저장장치도 파일 형태로 관리

---

### 3. 주요 디렉터리

| 디렉터리    | 설명               |
| ------- | ---------------- |
| `/`     | 최상위 루트 디렉터리      |
| `/home` | 사용자 계정 디렉터리      |
| `/root` | 관리자(root) 홈 디렉터리 |
| `/bin`  | 기본 실행 명령어 저장     |
| `/sbin` | 시스템 관리 명령어 저장    |
| `/etc`  | 시스템 설정 파일 저장     |
| `/dev`  | 장치 파일 저장         |
| `/tmp`  | 임시 파일 저장         |
| `/usr`  | 응용 프로그램 및 라이브러리  |
| `/var`  | 로그 및 가변 데이터 저장   |
| `/boot` | 부팅 관련 파일 저장      |

---

### 4. 파일의 종류

Linux에서는 모든 것을 파일로 취급한다.

| 종류      | 설명              |
| ------- | --------------- |
| 일반 파일   | 텍스트, 프로그램, 문서 등 |
| 디렉터리 파일 | 파일들을 관리하는 폴더    |
| 장치 파일   | 하드웨어 장치 표현      |
| 링크 파일   | 다른 파일을 참조       |
| 파이프 파일  | 프로세스 간 통신       |

---

### 5. 파일 권한 구조

Linux 파일은 소유자와 권한 정보를 가진다.

예시

```bash
-rwxr-xr--
```

구성

| 위치  | 의미        |
| --- | --------- |
| rwx | 소유자 권한    |
| r-x | 그룹 권한     |
| r-- | 기타 사용자 권한 |

권한 종류

| 문자 | 의미          |
| -- | ----------- |
| r  | 읽기(Read)    |
| w  | 쓰기(Write)   |
| x  | 실행(Execute) |

---

### 6. 파일 시스템 특징

* 계층적 디렉터리 구조 사용
* 모든 자원을 파일 형태로 관리
* 다중 사용자 환경 지원
* 강력한 권한 관리 기능 제공
* 장치도 파일처럼 접근 가능

---

### 정리

| 항목       | 내용                  |
| -------- | ------------------- |
| 루트 디렉터리  | `/`                 |
| 사용자 디렉터리 | `/home`             |
| 설정 파일    | `/etc`              |
| 프로그램 파일  | `/usr`, `/bin`      |
| 임시 파일    | `/tmp`              |
| 로그 파일    | `/var`              |
| 권한 종류    | 읽기(r), 쓰기(w), 실행(x) |
