### 1. VM (Virtual Machine)

#### 개요

VM(Virtual Machine)은 하나의 물리적 컴퓨터에서 여러 개의 운영체제를 독립적으로 실행할 수 있도록 해주는 가상화 기술이다.

#### 특징

* 하나의 PC에서 Windows, Linux 등 여러 OS 실행 가능
* 각 VM은 독립된 CPU, 메모리, 저장공간을 사용
* 실제 컴퓨터와 유사한 환경 제공
* 운영체제 전체를 설치해야 하므로 자원 사용량이 비교적 큼

#### 장점

* 운영체제 간 완전한 격리
* 테스트 및 개발 환경 구축에 용이
* 시스템 장애가 다른 VM에 영향을 주지 않음

#### 단점

* 메모리와 CPU 사용량이 많음
* 실행 속도가 상대적으로 느림

---

### 2. JVM (Java Virtual Machine)

#### 개요

JVM(Java Virtual Machine)은 Java 프로그램을 실행하기 위한 가상 실행 환경이다.

#### 특징

* Java 바이트코드(.class)를 실행
* 운영체제에 관계없이 동일한 Java 프로그램 실행 가능
* "Write Once, Run Anywhere" 지원

#### 동작 과정

1. Java 소스코드 작성(.java)
2. 컴파일(.class 생성)
3. JVM이 바이트코드를 해석 및 실행

#### 장점

* 높은 플랫폼 독립성
* 자동 메모리 관리(Garbage Collection)
* 안정성과 보안성 우수

#### 단점

* 네이티브 프로그램보다 실행 속도가 다소 느릴 수 있음

---

### 3. WSL2 (Windows Subsystem for Linux 2)

#### 개요

WSL2는 Windows에서 Linux를 가상 머신 없이 실행할 수 있도록 지원하는 기능이다.

#### 특징

* Windows 환경에서 Linux 명령어 사용 가능
* 실제 Linux 커널 사용
* Docker, 개발 도구, 서버 환경 구축에 적합
* Windows와 Linux 파일 시스템 연동 가능

#### 장점

* 별도의 듀얼부팅이나 VM 설치 불필요
* Linux 개발 환경을 쉽게 구성 가능
* 성능이 WSL1보다 향상됨

#### 단점

* 일부 GUI 프로그램은 추가 설정 필요
* 완전한 Linux 환경과는 약간의 차이가 존재

### 정리

| 구분    | VM         | JVM          | WSL2               |
| ----- | ---------- | ------------ | ------------------ |
| 목적    | 운영체제 가상화   | Java 프로그램 실행 | Windows에서 Linux 사용 |
| 실행 대상 | 운영체제 전체    | Java 바이트코드   | Linux 환경           |
| 특징    | 독립적인 OS 실행 | 플랫폼 독립성 제공   | Windows-Linux 통합   |
| 활용 분야 | 서버, 테스트 환경 | Java 애플리케이션  | 개발 및 서버 환경 구축      |

**한 줄 요약**

* **VM** : 운영체제를 가상으로 실행하는 기술
* **JVM** : Java 프로그램을 실행하는 가상 머신
* **WSL2** : Windows에서 Linux를 사용할 수 있게 해주는 환경
