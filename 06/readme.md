## 시스템 호출(System Call) 및 파일 디스크립터(File Descriptor)

### 1. 시스템 호출(System Call)

#### 개요

시스템 호출(System Call)은 사용자 프로그램이 운영체제의 기능을 사용할 수 있도록 제공되는 인터페이스이다. 프로그램이 직접 하드웨어에 접근하지 않고 운영체제를 통해 필요한 작업을 수행한다.

#### 주요 역할

* 파일 생성, 읽기, 쓰기, 삭제
* 프로세스 생성 및 종료
* 메모리 관리
* 입출력 장치 제어

#### 대표적인 시스템 호출

| 함수        | 설명      |
| --------- | ------- |
| `open()`  | 파일 열기   |
| `read()`  | 파일 읽기   |
| `write()` | 파일 쓰기   |
| `close()` | 파일 닫기   |
| `fork()`  | 프로세스 생성 |
| `exec()`  | 프로그램 실행 |
| `exit()`  | 프로세스 종료 |

#### 동작 과정

1. 사용자 프로그램이 시스템 호출 요청
2. 운영체제 커널로 제어권 이동
3. 커널이 요청된 작업 수행
4. 결과를 프로그램에 반환

---

### 2. 파일 디스크립터(File Descriptor)

#### 개요

파일 디스크립터(File Descriptor)는 운영체제가 파일이나 입출력 장치를 관리하기 위해 사용하는 정수 형태의 식별자이다.

파일을 열면 운영체제가 파일 디스크립터를 할당하며, 이후 파일 입출력 작업은 이 번호를 통해 수행된다.

---

### 3. 기본 파일 디스크립터

Linux에서는 프로세스가 시작될 때 기본적으로 다음 3개의 파일 디스크립터를 가진다.

| 번호 | 이름     | 설명         |
| -- | ------ | ---------- |
| 0  | stdin  | 표준 입력(키보드) |
| 1  | stdout | 표준 출력(화면)  |
| 2  | stderr | 표준 에러 출력   |

예시

```c
write(1, "Hello\n", 6);
```

→ 파일 디스크립터 1(stdout)을 통해 화면에 출력한다.

---

### 4. 파일 디스크립터 사용 예

```c
#include <fcntl.h>
#include <unistd.h>

int main() {
    int fd;

    fd = open("test.txt", O_RDONLY);

    close(fd);

    return 0;
}
```

설명

* `open()` : 파일을 열고 파일 디스크립터 반환
* `fd` : 파일 디스크립터 저장
* `close()` : 파일 닫기

---

### 5. 시스템 호출과 파일 디스크립터의 관계

파일 관련 시스템 호출은 파일 디스크립터를 사용하여 파일을 관리한다.

예시

```c
fd = open("data.txt", O_RDONLY);
read(fd, buffer, 100);
close(fd);
```

동작 순서

1. `open()`으로 파일 열기
2. 파일 디스크립터 반환
3. `read()`로 파일 읽기
4. `close()`로 파일 닫기
