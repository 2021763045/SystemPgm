## 하드 링크(Hard Link)와 심볼릭 링크(Symbolic Link) 정리

### 1. 개요

링크(Link)는 하나의 파일을 여러 이름으로 참조할 수 있도록 해주는 기능이다. Linux에서는 **하드 링크(Hard Link)** 와 **심볼릭 링크(Symbolic Link)** 를 제공한다.

---

### 2. 하드 링크(Hard Link)

#### 개념

하드 링크는 원본 파일과 동일한 데이터를 가리키는 또 다른 파일 이름이다. 원본 파일과 하드 링크는 같은 inode 번호를 공유한다.

생성 방법

```bash
ln 원본파일 링크파일
```

예시

```bash
ln file.txt hardlink.txt
```

#### 특징

* 원본 파일과 동일한 데이터 사용
* 같은 inode 번호 공유
* 원본 파일을 삭제해도 데이터 유지
* 동일한 파일 시스템 내에서만 생성 가능
* 디렉터리에는 일반적으로 생성 불가

#### 장점

* 저장 공간 추가 사용 없음
* 원본 삭제 시에도 데이터 유지

#### 단점

* 다른 파일 시스템에서는 생성 불가
* 관리가 다소 복잡함

---

### 3. 심볼릭 링크(Symbolic Link)

#### 개념

심볼릭 링크는 원본 파일의 경로를 저장하는 특수한 파일이다. Windows의 바로가기(Shortcut)와 유사하다.

생성 방법

```bash
ln -s 원본파일 링크파일
```

예시

```bash
ln -s file.txt symlink.txt
```

#### 특징

* 원본 파일 경로를 참조
* 별도의 inode 사용
* 파일과 디렉터리 모두 링크 가능
* 다른 파일 시스템에도 생성 가능
* 원본 삭제 시 링크가 깨짐(Broken Link)

#### 장점

* 사용이 편리함
* 디렉터리 링크 가능
* 다른 파일 시스템에서도 사용 가능

#### 단점

* 원본이 삭제되면 사용할 수 없음

---

### 4. 하드 링크와 심볼릭 링크 비교

| 구분          | 하드 링크     | 심볼릭 링크   |
| ----------- | --------- | -------- |
| inode 공유    | O         | X        |
| 원본 삭제 시     | 계속 사용 가능  | 링크 깨짐    |
| 파일 시스템 간 생성 | X         | O        |
| 디렉터리 링크     | X         | O        |
| 저장 방식       | 실제 데이터 참조 | 경로 정보 참조 |

---

### 5. 링크 확인

명령어

```bash
ls -l
```

출력 예시

```text
lrwxrwxrwx 1 user user 8 Mar 1 symlink.txt -> file.txt
```

* `l` : 심볼릭 링크
* `->` : 연결된 원본 파일 표시

---

### 정리

| 종류     | 특징                           |
| ------ | ---------------------------- |
| 하드 링크  | 같은 inode 공유, 원본 삭제 후에도 사용 가능 |
| 심볼릭 링크 | 경로를 참조하는 바로가기 형태             |
| 생성 명령어 | `ln`, `ln -s`                |
| 확인 명령어 | `ls -l`                      |
